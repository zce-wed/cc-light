# 快捷记录 · 历史搜索 设计

- 日期:2026-07-15
- 范围:cc-light 快捷记录的历史窗口
- 状态:设计待审

## 背景

cc-light 的快捷记录由「悬浮便签」+「历史窗口」组成:

- **便签**:多行输入,`↻` 记录并清空、`☰` 打开历史、`—` 最小化、`✕` 关闭。
- **历史(`☰`)**:最新在上,左键回填便签、右键删除;窗口跟便签走、高度按条目自适应、滚动条按需。
- **数据**:`notes_history.json`(数组,末尾最新,上限 `MAX_HISTORY=500` 条)。

用户主要用快捷记录记**待办/任务**和**工作备忘**。记录多了之后,**最痛的是"找不到旧记录"**,找的时候靠**内容关键词**回忆。

## 目标

让历史窗口支持**全文关键词搜索 + 命中高亮**,快速定位旧记录。

## 设计

### 1. UI 与交互

历史窗口从上到下:

1. **标题行** — 有搜索时 `快捷记录历史 · 匹配 X / 共 Y 条`;无搜索时 `快捷记录历史(共 Y 条)`。
2. **搜索框** — `Entry`,灰字占位「搜索…」,首次获得焦点清空并转黑字。
3. **列表区** — 现有 Canvas + 按需滚动条,保持不变。

行为:

- **输入即筛**(不区分大小写、子串匹配),无需回车;清空搜索框 → 恢复全部。
- 每条记录的预览从 `Label` 换成**只读 `Text`**(`state="disabled"`);命中关键词以黄底高亮。
- 左键点记录 → 回填便签(不变);右键 → 删除(不变)。
- 搜索框聚焦时按 `Esc` → 清空搜索,恢复全部。

### 2. 筛选与预览逻辑

**匹配规则** — `query.lower() in text.lower()`,全文子串、不区分大小写;空 query 显示全部。

**预览动态片段**(高亮有用性的关键):

- 无搜索:预览 = 前 14 字符 + `…`(保持现状)。
- 有搜索:定位命中词在全文的位置,预览改为**命中片段**(命中词前后各 ~8 字符,头尾按需补 `…`),命中词黄底高亮;多命中取第一个。
- 例:全文 `今天要改 cc-light 的搜索功能`,搜「搜索」→ 预览 `…cc-light 的【搜索】功能`,【】处高亮。

**显隐策略** — 打开历史时一次性创建所有 row(与现状一致);搜索时对匹配的 `pack`、不匹配的 `pack_forget`,**不重建 widget**;仅对可见 row 重设预览内容 + 高亮 tag。

**标题计数** — 有搜索:`匹配 X / 共 Y 条`;无搜索:`共 Y 条`。无匹配 → 列表区显示「无匹配记录」。

### 3. 边界与实现要点

**改动范围** — 仅 `show_history` 函数:加搜索框 `Entry`、标题计数、预览换 `Text`、筛选/高亮逻辑。`notes_history.json` 结构不变;便签、灯窗口、菜单都不动。

**实现要点**:

- 预览 `Text`:`state="disabled"` 防误输入;仍绑 `<Button-1>` 回填 / `<Button-3>` 删除(disabled 不挡鼠标事件);高亮 `tag_configure("hl", background=...)` + `tag_add`,disabled 下可用。
- placeholder:Entry 初始灰字「搜索…」+ 标记位;`<FocusIn>` 首次清空转黑字;筛选时若处 placeholder 态则视为空 query。
- `Esc`:搜索框 `<Escape>` → 清空 + 恢复全部。

**边界**:

- 空历史(无记录):不显示搜索框,只显示「(无记录)」。
- query 含特殊字符:纯子串字面匹配(非正则),无注入风险。
- 多行文本:`flat = " ".join(text.split())` 压平后再匹配/取片段(沿用现有)。
- 无匹配:列表区「无匹配记录」。

## 组件与数据流

```
notes_history.json  --load_history()-->  items(全量,倒序)
        |
   [用户在搜索框输入 query]
        v
   match_record / preview_snippet(纯函数)
        |
   pack/pack_forget row + 重设可见 row 的 Text 预览(片段)+ 高亮 tag
        |
   标题更新(匹配 X / 共 Y)
```

抽两个纯函数(便于测试,不依赖 tkinter):

- `match_record(text, query) -> bool`:不区分大小写子串匹配。
- `preview_snippet(text, query, head=14, ctx=8) -> (snippet, spans)`:返回预览片段与命中区间列表(用于高亮)。空 query → `(text 压平后前 head 字 + ' …' 若超长, [])`。

## 错误处理

- 数据读取/解析异常:沿用现有 `try/except`,失败按空列表处理。
- 搜索/片段计算异常:`try/except` 兜底,失败退化为"匹配但不高亮"。
- Text 内容重设异常:单条失败不影响其他条目。

## 验证计划

cc-light 无测试框架,两层验证:

1. **纯函数单测**(小脚本,不依赖 tkinter),跑边界用例:
   - 空 query → 全部匹配,片段为前 14 字。
   - 命中在首 / 中 / 尾。
   - 大小写不敏感。
   - 多行文本(压平后匹配)。
   - query 比文本长 → 不匹配。
   - 无命中 → 不匹配。
2. **端到端手动**:重启灯窗口 → `☰` → 输词 → 验筛选 / 高亮 / 计数 / 清空恢复 / Esc / 左键回填 / 右键删除。
